# 文件传输专项规划 — 打造核心竞争优势

> 定位：文件传输是 TermAI 相对 SecureCRT（+SecureFX）的头号差异化能力。
> 目标：比 SecureFX 更快、比 WinSCP 更顺手、和终端与 AI 深度融合。

## 1. 竞品痛点分析

| 竞品 | 痛点 |
|------|------|
| SecureCRT + SecureFX | 传输是独立付费软件，切换割裂；UI 老旧；无智能重试策略展示 |
| WinSCP | 与终端割裂（要单独开）；队列管理繁琐；无 AI |
| Xftp | 商业授权限制；与 Xshell 联动仍是两个窗口 |
| lrzsz (Zmodem) | 单文件、慢、无进度可靠性，仅作兼容手段 |

我们的机会：**终端、文件、AI 在同一个窗口内闭环**。

## 2. 能力蓝图（三阶段）

### Phase 1 — 传输引擎 v1（本期交付）
- **统一传输队列**：所有上传/下载进队列，全局传输面板集中展示
- **实时进度**：字节数、百分比、瞬时速率、ETA，100ms 节流推送
- **断点续传**：目标文件已存在且小于源文件时，自动从断点 offset 续传；
  等大时直接跳过（目录同步式增量）
- **目录递归传输**：整个目录上传/下载，自动创建层级，逐文件汇报
- **可取消**：任意传输随时取消，不影响队列中其他任务
- **拖拽上传**：把文件/文件夹拖进 SFTP 面板即传到当前目录

### Phase 2 — 可靠与效率（下期）
- **完整性校验**：传输后 SHA-256 比对（远端执行 sha256sum / 本地计算）
- **自动重试**：网络抖动自动重连续传（指数退避，最多 N 次）
- **限速与并发度**：全局/单任务限速，多文件并行（默认 2–4 并发）
- **队列持久化**：应用重启后未完成任务可恢复
- **传输历史**：成功/失败记录、审计日志
- **冲突策略**：覆盖/跳过/重命名/仅更新，批量应用

### Phase 3 — 智能与生态（远期）
- **双向目录同步**：类 rsync 增量同步，预览差异后执行
- **Zmodem 兼容**：终端内 rz/sz 直接接管（老运维习惯）
- **AI 传输助手**：自然语言驱动——"把 /var/log 里最近 3 天的日志打包下载"
  → AI 生成计划（远端 tar → 下载 → 清理）→ 用户确认执行
- **终端联动**：终端里 `ls` 输出的文件名可右键直接下载

## 3. 技术设计（Phase 1）

```
前端 SftpView / 拖拽 ──> transfer_start(kind, local, remote)
                              │ Channel 事件流
Rust transfer.rs：
  TransferManager { id -> CancelFlag }
  执行器（tokio task）：
    1. 探测源类型（文件/目录），统计总字节数与文件数 → Started
    2. 逐文件：
       - 目标存在且 size == 源 → 跳过（Skipped）
       - 目标存在且 size < 源  → 双端 seek 到 offset，续传
       - 否则从 0 传输
       - 64KB 分块循环拷贝，每块检查取消标志，节流发 Progress
    3. 完成 → Done / 出错 → Error / 取消 → Cancelled
```

- 断点续传实现基础：russh-sftp `File` 实现 AsyncSeek，
  `open_with_flags(OpenFlags::READ/WRITE/CREATE/APPEND)` 控制打开模式
- 进度速率：滑动窗口（最近 2s）计算瞬时速率与 ETA
- 前端：全局底部传输抽屉（TransferPanel），标签栏徽标显示活跃传输数

## 4. 体验红线

- 传输过程绝不阻塞终端操作（引擎全部在 Rust 异步任务中）
- 任何失败必须给出可读的中文原因与续传/重试入口
- 拖拽、右键、按钮三种入口等价，路径永远可见可编辑
